# Caption CLI Command Map

This file is a routing map, not a manual. The authoritative contract is generated by the CLI itself:

```bash
caption capabilities        # machine-readable: usage, notes, examples, exit codes, env vars
caption robot-docs guide    # markdown agent handbook from the same command table
caption <command> --help    # per-command syntax
```

Never trust this file over live output. If a flag here disagrees with `capabilities`, the CLI wins.

## Invocation shape

```bash
caption <global-flags> <command> [command-flags]
```

Global flags (before the subcommand):
- `--output json|table|md` - per-command defaults vary; use `json` for parsing
- `--output-file <path>` - write rendered output to a file; use `caption_cache/` for every request
- `--env-file <path>` - dotenv loaded before env resolution (default `$PWD/.env`)
- `--cache-path <path>` - search-token cache location

`caption --version` is unsupported; read `capabilities.version`.

## Command groups

### Self-description (no network, no credentials)
- `capabilities` - full CLI contract as JSON
- `robot-docs guide` - agent handbook as Markdown

### Health
- `doctor [--strict]` - probes Caption API core + hosted history support; `--output json` gives `{organization, features, probes}`; `--strict` exits 1 on probe failure

### Search (needs CAPTION_API_URL, CLERK_API_KEY, CAPTION_MEILI_URL)
- `token [--show-token]` - fetch/cache Meili credentials; redacted by default
- `search <query> [--index INDEX] [--limit N] [--show-dupes]` - default index `transcript_captions_v1`; auto-refreshes token once on auth failure

### Workspace (needs CAPTION_API_URL, CLERK_API_KEY)
- `list_projects [--full]` - condensed by default; `items[].transcript` holds the transcript UUID for `dl_transcript`
- `list_folders [--full]`
- `create_project <name> [--description] [--workspace-id] [--dry-run]`
- `create_folder <name> [--description] [--parent] [--workspace-id] [--dry-run]`
- `edit_project <id> [--name] [--description|--clear-description] [--folder|--clear-folder] [--dry-run]` - requires >=1 field; set/clear pairs conflict
- `edit_folder <id> [--name] [--description|--clear-description] [--parent|--clear-parent] [--dry-run]` - same validation

### Transcripts and speakers (needs CAPTION_API_URL, CLERK_API_KEY)
- `dl_transcript <transcript_id> [--timestamp]` - timestamps stripped by default
- `tail [transcript_id] [--duration SECS] [--max-events N] [--idle-timeout SECS] [--clerk-api-key TOKEN]` - streams finalized captions (full backfill, then follows live); omitted ID = transcript of most recently updated project; plain output `{channel}-{index}: {content}`, diagnostics on stderr; run in background with stdout redirected to `caption_cache/`
- `list_speakers <transcript_id>` - derives (channel, index, speakerId) groups; speaker IDs only, no names
- `assign_speakers (--transcript-id|--project-id) --channel {0|1|2|microphone|loopback|external} [--index N] (--speaker-id|--name) [--dry-run]` - prefer `--name`; omitting `--index` hits all indexes in the channel; `--project-id` fans out over all project transcripts
- `rename_speaker <project_id> <speaker_id> --name TEXT [--dry-run]` - custom speakers only

### Hosted history / agentsview (needs CLERK_API_KEY, ORGANIZATION_ID; also accepts --clerk-api-key / --org-id)
- `list_matters [--include-one-shot] [--full]` - hosted server projects; NOT the same as `list_projects`
- `sync --session-id VALUE [--project-name] [--test|--dry-run] [--yes] [--db-path]` - substring match on session ID; `'*'` without `--test` requires `--yes`; exit 4 if any session fails
- `create_md <file> [--project-id|--project-name] [--title] [--dry-run]` - `--project-id` wins if both selectors given
- `list_md [--project] [--exclude-project] [--tag] [--created-by] [--sort recent|most_starred] [--cursor] [--limit] [--full]`
- `get_md <id>` - saves markdown to `--cache-dir/md/{title}.md` unless `--output-file` given

## Exit codes

- 0 - success (including empty results)
- 1 - user-input error; also `doctor --strict` probe failure
- 2 - usage error (argparse: unknown command/flag, missing argument)
- 3 - configuration error (missing required env var for that command)
- 4 - upstream failure (HTTP/Meili error, malformed server response)
- 5 - remote resource not found (HTTP 404)
